# Context or Tuning? Layer-Level Analysis of Embedding Degradation in Clinical Document Retrieval

[![License: CC BY 4.0](https://img.shields.io/badge/License-CC%20BY%204.0-lightgrey.svg)](https://creativecommons.org/licenses/by/4.0/)

Code, data, and results for the paper:

> Mikkelsen Y. Context or Tuning? Layer-Level Analysis of Embedding Degradation in Clinical Document Retrieval. Submitted to *Journal of the American Medical Informatics Association (JAMIA)*. 2026.

## Overview

This study extracts embeddings from all hidden layers of 9 transformer models (11 configurations) across 3 clinical corpora to investigate where embedding degradation occurs, whether it is driven by model tuning or input context, and whether post-hoc corrections can recover retrieval performance without retraining.

Key findings:
- All models exhibit a U-shaped MRR@10 curve across layers, collapsing at intermediate depths
- Final-layer recovery is driven by geometric restructuring (participation ratio expansion) learned through contrastive training
- Model rankings are stable across two independent validation corpora of different clinical document types (cross-source ρ = 0.976)
- ZCA whitening provides the largest post-hoc gains; corpus-only whitening is effective on heterogeneous clinical text

## Corpora

**Primary experiment (100 documents each):**
- MTSamples medical transcriptions (first 100 rows of `mtsamples_sample.csv`)
- PMC-Patients case reports (sampled from [zhengyun21/PMC-Patients](https://huggingface.co/datasets/zhengyun21/PMC-Patients), seed=42)
- Synthetic clinical notes (generated by Mistral-7B-Instruct)

**Validation corpora:**
- 500 PMC-Patients (seed=123, no overlap with primary sample)
- 400 MTSamples (remaining documents after excluding primary 100)

Queries (100 keyword + 100 natural language per corpus) were generated from document metadata using GPT-4o, as described in the [companion study](https://github.com/yngvemikkelsen/clinical-rag-retrieval-benchmark).

## Reproducing Results

### Requirements

```bash
pip install -r requirements.txt
```

Tested on Google Colab Pro+ with NVIDIA H100 80GB. BERT-scale models run in fp32; LLM-scale models in fp16.

### Step 1: Main Experiment

```bash
python scripts/paper2_layer_analysis.py
```

Extracts all hidden layers from all models, computes MRR@10, Recall@10, participation ratio, and average cosine similarity at each layer. Outputs per-model checkpoints to `results/`. Runtime: ~30 minutes on H100.

### Step 2: Supplementary Experiments

```bash
python scripts/paper2_supplementary.py
```

Runs whitening variant comparison (transductive, corpus-only, held-out) and validation corpus analysis. Outputs to `supplementary/`. Runtime: ~2 hours on H100.

### Step 3: Figures

```bash
python scripts/paper2_figures.py
```

Generates all 6 main figures + 1 supplementary figure in PDF and PNG format.

## Data Notes

- **PMC-Patients** is loaded from HuggingFace. The exact samples are reproducible via the seeds specified in the scripts (seed=42 for primary, seed=123 for validation).
- **MTSamples** (`mtsamples_sample.csv`) is included in the [companion study repository](https://github.com/yngvemikkelsen/clinical-rag-retrieval-benchmark).
- **Synthetic clinical notes** and **metadata queries** are also in the companion study repository.
- Result parquets in `results/` and `supplementary/` contain all computed metrics and can be used directly without re-running the experiments.
